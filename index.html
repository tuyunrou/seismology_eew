板塊構造

臺灣東部海域菲律賓海板塊向北隱沒到歐亞大陸板塊下，南部海域歐亞大陸板塊向東隱沒到菲律賓海板塊下。

災害風險

1900 年以來共發生 7 次大規模災害地震（罹難百人以上），每年地震造成數百人傷亡與大量房屋損失。

強震即時警報原理

地震發生時會同時產生 P 波（縱波）與 S 波（橫波），P 波傳播速度快震幅小，藉由 P 波資訊推估地震大小，於災害性 S 波抵達前發布警報。

人工智慧與機器學習

從傳統演算法到自監督學習 — 地震監測技術的典範轉移

🎯

無標註學習

無需人工標記正確答案，直接利用數據本身的結構進行訓練。

🧠

通用表示

學習具有信息量且通用的特徵表示 (Representations)，可遷移至多種下游任務。

⚡

高效微調

只需少量任務導向數據即可達到 State-of-the-art 性能，大幅減少標註成本。

SeisWav2Vec 2.0 — 從語音到地震

延續 Wav2Vec 2.0 在語音處理上的成功，將其核心思想遷移至地震波形分析。基於對比學習 (Contrastive Learning) 的自監督模型，旨在區分正確樣本與干擾項。

📊

大型地震模型 (LEM)

地震監測的新紀元，透過大規模預訓練建構通用地震波形理解模型。

🔬

波相辨識突破

傳統演算法需大量參數調整且準確性有限；深度學習大幅提升 P 波與 S 波到時判別精度。

💡

預訓練 + 微調

在預訓練之後接上任務導向的 CNN 解碼器，採用三種策略進行微調，適應不同監測任務。

⚠️

傳統演算法限制

於自動辨識地震波相上有大量參數需調整，且準確性難以提升。

🗄️

標註資料稀缺

深度學習需要海量標註數據。地震台站產生大量數據，但多數未經標註，形成資源浪費。

🚀

自監督學習解方

自監督學習能從未標註資料中提取特徵，突破傳統瓶頸，讓大量未標註資料發揮價值。

AI 代理群 — 多代理協作系統

整合地震監控、自動化派遣與科研分析的閉環系統，四位專責代理協同運作。

🤖

my_agent

預警總管 · 監控與警報核心

維護監控 Cron、異常推播與日報總結。每 30 分鐘執行健康檢查，異常時推送最後 50 行日誌至 Telegram。

  • EEW 流程監控與 .rep 分析
  • 延遲量測與異常告警
  • 協調各 Agent 任務分派
  • 每日摘要彙編
🔄

secondary_agent

整合工程師 · 自動化調度中心

負責 Hermes 鏈路轉發與跨代理任務派遣。開發 seismic-conductor 模組，實現標準化架構。

  • 跨 Agent 互叫機制
  • 結構化數據路由分發
  • Host-side 腳本維護
  • Cron 配置與 Telegram 推播
📊

seismo_agent

地震學助教 · 數據科學流水線

解析 .rep 檔案、效能延遲量測與自動繪圖。原始檔轉為 CSV/JSON，自動生成散布與序列圖。

  • .rep 轉換與結構化輸出
  • 圖表自動生成
  • 雙重存檔 (二進位 + 數據)
  • Docker 容器健康管理
🔬

seismo

科學研究員 · 數據建模

專注地震數據建模、實驗設計與趨勢預測。針對密集小震群與深部無感地震建立統計模型。

  • 長期趨勢分析
  • 預警閾值動態調整建議
  • 科學驗證報告產出
  • 學術論文數據鏈路追蹤

科技極限與挑戰

⚠️

誤報風險

短時間內發生兩個地震可能造成誤報。如 111 年 9 月 18 日雲嘉南地區誤發國家級警報，及 115 年 2 月 23 日馬來西亞婆羅洲外海地震導致誤發中小學 EEW。

🔇

漏報挑戰

兩個地震相差 4 秒時，系統僅處理第一個地震，後面較大的地震可能被漏報。

🏗️

複合型災害

深震、海嘯、強震持續時間等因素交織，須同時考慮強度與持續時間。

AI × Workflow 技術演進

2022 → 2026，用 2026 年人類思維的方式去思考每一項工作

2022 – 2024

對話時代

ChatGPT 登場，寫程式、查指令、翻譯、寫英文信都用「問」的。產出仍需自己手動複製、貼上、在本機執行。AI 是助理，人類是手腳。

2024 – 2025

整合介面

Agent 進入 IDE：Cursor、Windsurf、GitHub Copilot Agent Mode 等 AI 原生編輯器登場。能直接改多檔程式、執行終端機指令、跑測試、部署到雲端。AI 開始接管「執行」那一段工作。

2025 – 2026

自主 Agent

模型能力持續增強，專門化 agent 出現。Claude Code 等 CLI agent 能獨立完成多步驟任務。思維轉換：別再問「我要怎麼做這件事」，改問「要交給哪個 agent、怎麼驗收？」

VS Code — 貫穿三個階段的基座平台：Copilot Chat 住在這，Cursor / Windsurf / Antigravity 都是它的 fork。
